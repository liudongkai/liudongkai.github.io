иҜҙжҳҺ: Sets environment variables based on attributes of the request without respect to case
иҜӯжі•: SetEnvIfNoCase attribute regex [!]env-variable[=value] [[!]env-variable[=value]] вҖҰ
дёҠдёӢж–Ү: жңҚеҠЎеҷЁй…ҚзҪ®, иҷҡжӢҹдё»жңә, зӣ®еҪ•, .htaccess
иҰҶзӣ–йЎ№: FileInfo
зҠ¶жҖҒ: Base
жЁЎеқ—: mod_setenvif
е…је®№жҖ§: Apache 1.3 and above

The SetEnvIfNoCase is semantically identical to the SetEnvIf directive, and differs only in that the regular expression matching is performed in a case-insensitive manner. For example:
SetEnvIfNoCase Host Apache.Org site=apache
This will cause the site environment variable to be set to вҖңapacheвҖқ if the HTTP request header field Host: was included and contained Apache.Org, apache.org, or any other combination.
SetEnvIfNoCase еҪ“ж»Ўи¶іжҹҗдёӘжқЎд»¶ж—¶пјҢдёәеҸҳйҮҸиөӢеҖјпјҢдёҖиҲ¬з»“еҗҲе…¶д»–жҢҮд»ӨдҪҝз”ЁгҖӮ
